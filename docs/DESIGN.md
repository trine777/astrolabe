# 星图 (XingTu) 总体设计

> 版本: v0.1.0 | 归属: 星空座 | 状态: 设计中

## 1. 系统定位

### 1.1 核心定位

> **星图 = 星空座的"数据世界模型"工具**
>
> 让用户能把 CSV 数据 → 压缩成语义知识 → 沉淀到星空座 → 产出可复用的数据胞式与行技。

### 1.2 设计原则

1. **闭环优先**：每个功能都要形成"试炼 → 成型 → 回补"的闭环
2. **器官协作**：遵循五器官分工，不越界不混淆
3. **可扩展**：数据源、AI 后端、查询引擎均可插拔
4. **风隐美学**：UI 遵循深色主题、科技仙侠风格

## 2. 风隐 OS 架构定位

```
上界：知识层（GitHub / 文档 / 经验 → 风隐知识铸块）
-----------------------------------------
器官层（五器官）      ← 星图元数据归属星空座
胞式层               ← 星图产出数据胞式
行技层               ← 星图产出数据行技
行阵 / 局谱 / 膜
-----------------------------------------
内城：Flow City      ← 星图的试炼场（UI 在此）
-----------------------------------------
外城：风港           ← 星图能力的商业化出口
-----------------------------------------
下界：行械层         ← 星图调用的执行器（L0 机械钉）
```

## 3. 五器官协作

| 器官 | 角色 | 在星图中的职责 |
|------|------|----------------|
| **星空座** | 本体归属 | 元数据存储、世界模型、语义图谱、指标定义、范式沉淀 |
| **语界枢** | 输入通道 | UI 采集、自然语言解析、结构化转换、API 接口 |
| **影澜轩** | 变化感知 | 事件流、变更审计、数据解释、洞察生成 |
| **序律腺** | 节奏调度 | 数据刷新计划、ETL 调度、行技编排 |
| **炽心核** | 执行意志 | 查询执行、分析决策、能量消耗计量 |

## 4. 操作循环（完整闭环）

```
1. 现实任务
   └─ 用户有一批 CSV 业务数据需要分析

2. Flow City 试炼
   └─ 打开星图界面，拖入 CSV 文件

3. 熔心炉压缩
   ├─ AI 推断列的语义类型
   ├─ 用户确认/修改元数据
   ├─ 定义业务指标
   └─ 建立数据关系

4. formed 成型
   ├─ 元数据写入星空座
   ├─ 胞式/行技注册到 OS
   └─ 模板沉淀到 formed

5. 行技调用
   ├─ 自然语言查询数据
   ├─ 跨数据源联动分析
   └─ 生成可视化报告

6. 现实结果
   ├─ 业务洞察
   └─ 商业价值

7. 回补进化
   ├─ 影澜轩记录洞察
   ├─ 星空座沉淀模式
   └─ 元数据模板复用

8. 下一轮更强
   ├─ 新数据导入更快
   ├─ 关系推断更准
   └─ 分析能力更强
```

## 10. UI 设计要点

遵循风隐美学（询风台 UI 设计规范）：

- **配色**：风蓝 `(0.4, 0.6, 0.9)` / 星紫 `(0.6, 0.5, 0.8)` / 炽金 `(0.9, 0.7, 0.5)`
- **背景**：深邃星空渐变
- **风格**：科技仙侠感、统一深色主题

### 核心界面

1. **拖放区域**：支持多文件拖入，显示解析进度
2. **元数据编辑器**：表格形式展示属性，区分 AI 推断和用户确认
3. **数据浏览器**：预览数据，支持筛选排序
4. **关系图**：力导向图展示对象关系
5. **事件时间线**：影澜轩变更历史

## 11. 后续扩展路线

1. **数据源扩展**：JSON、Excel、API、数据库直连
2. **AI 后端扩展**：Claude、本地 LLM（Ollama）
3. **formed 模板库**：行业数据模板沉淀
4. **风港商业化**：元数据服务、数据分析 SaaS
5. **深度器官集成**：
   - 序律腺：数据刷新调度
   - 炽心核：分析任务优先级
   - 影澜轩：数据血缘追踪
